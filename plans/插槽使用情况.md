# Vue 插槽使用情况

> 项目中 Vue 插槽 (Slots) 的使用情况记录，包括已实现的优秀实践和后续可改进方向

## ✅ 已正确使用插槽的组件

### 1. [`EventTabs.vue`](../frontend/src/components/HomeEventAdmin.vue/Event/EventTabs.vue)
- 使用命名插槽：`teams`、`group`、`knockout`、`schedule`
- 父组件通过 `<template #xxx>` 注入标签内容
- ✅ 优秀实践

### 2. [`ProfileCard.vue`](../frontend/src/components/MyProfile/ProfileCard.vue)
- 使用 `默认插槽` + `actions` 命名插槽
- 支持加载中/加载失败/空数据状态 + 自定义内容
- ✅ 良好设计

### 3. [`HomeLayout.vue`](../frontend/src/components/General/HomeLayout.vue)
- 使用 `sidebar` 命名插槽 + `默认插槽`（作用域插槽传递 `activeTab`）
- `#header-nav` 插槽透传到 `HomeHeader`
- ✅ 良好设计

### 4. [`HomeHeader.vue`](../frontend/src/components/General/HomeHeader.vue)
- 使用 `logo`、`header-prepend`、`header-append`、`nav` 命名插槽
- ✅ 灵活设计

### 5. [`PageHeader.vue`](../frontend/src/components/General/PageHeader.vue)
- 使用 `title`、`extra`、`actions` 命名插槽
- ✅ 良好抽象

### 6. [`AsyncContent.vue`](../frontend/src/components/General/AsyncContent.vue)
- 使用 `默认插槽` + `retry`、`empty` 命名插槽
- 统一处理 loading/error/empty 状态
- ✅ 已消除大量重复代码

### 7. Element Plus 插槽使用
- [`el-alert`] 的 `#actions` 插槽 — 多处使用
- [`el-popconfirm`] 的 `#reference` 插槽 — 多处使用
- [`el-dialog`] 的 `#footer` 插槽 — 多处使用
- ✅ 标准用法

## 🟡 已识别但未实施的插槽改进机会

### 1. [`EventToolbar.vue`](../frontend/src/components/HomeEventAdmin.vue/Event/EventToolbar.vue)
当前硬编码了"邀请球队"、"删除赛事"、"添加赛事"三个按钮。可以通过插槽支持自定义按钮。

```vue
<template>
  <div class="area-header">
    <div class="header-left">
      <h2 class="area-title">我的赛事</h2>
      <slot name="selector" />
    </div>
    <div class="header-actions">
      <slot name="actions" />
    </div>
  </div>
</template>
```

### 2. [`HomeTabs.vue`](../frontend/src/components/General/HomeTabs.vue)
当前通过 props 传入菜单配置实现复用，可进一步提升为插槽形式：

```vue
<HomeTabs>
  <template #items>
    <el-menu-item index="events">赛事</el-menu-item>
    <el-menu-item index="teams">球队</el-menu-item>
  </template>
</HomeTabs>
```

### 3. [`EventGroupCard.vue`](../frontend/src/components/HomeEventAdmin.vue/Event/EventGroupCard.vue)
卡片头部操作按钮可以插槽化：

```vue
<div class="group-card-header">
  <div class="group-header-left">
    <slot name="header-left">
      <span class="group-name">{{ groupStage.name }}</span>
      <div class="group-info-tags">...</div>
    </slot>
  </div>
  <div class="group-header-actions">
    <slot name="header-actions">
      <!-- 默认取消分组/分配球队按钮 -->
    </slot>
  </div>
</div>
```

### 4. [`EventGroupAssign.vue`](../frontend/src/components/HomeEventAdmin.vue/Event/EventGroupAssign.vue)
1000+ 行的超大对话框可将部分区域插槽化：
- 顶部的 `#scope-bar` 插槽
- 提示文字 `#hint` 插槽
- 底部的 `#footer-extra` 插槽（在确认按钮前插入额外按钮）

### 5. [`AddEvent.vue`](../frontend/src/components/General/AddEvent.vue)
表单字段可以使用插槽支持扩展：

```vue
<slot name="form-fields" />
<!-- 默认表单字段 -->
```
